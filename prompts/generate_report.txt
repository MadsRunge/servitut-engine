Ejendommen omfatter disse matrikler: {all_matrikler_json}
Rapportens datoafgrænsning er: {as_of_date}

Din opgave er at vurdere ALLE servitutter og afgøre om de vedrører projektområdet.
Inkludér alle servitutter i outputtet — også Nej og Måske — sorteret: Ja → Måske → Nej.

Returner dit svar som et JSON-objekt med disse felter:

- entries: Array af redegørelses-poster, én pr. servitut, med felterne:
  - nr: Løbenummer (1, 2, 3...)
  - date_reference: Dato/løbenummer fra kilden
  - description: Præcis beskrivelse af servituttens indhold (2-4 sætninger)
  - beneficiary: Påtaleberettiget
  - disposition: "Rådighed" eller "Tilstand"
  - legal_type: "Offentligretlig" eller "Privatretlig"
  - action: Anbefalet handling
  - scope: "Ja" hvis servitutten gælder mindst én af projektmatriklerene | "Nej" hvis den ikke gælder nogen | "Måske" hvis uafklaret
  - scope_detail: Kort tekst der beskriver præcist hvilke matrikler servitutten vedrører, fx "Vedr. matr.nr. 1o og 1v" eller "Vedrører ikke projektområdet (vedr. matr.nr. 38b)" eller "Uafklaret — matrikel ikke nævnt i aktteksten"
  - relevant_for_project: true hvis scope er "Ja" eller "Måske" og servitutten er byggerelevant
  - servitut_id: ID fra input-data

- notes: Generelle bemærkninger om servitutsituationen (valgfrit)

VIGTIGT:
- Brug applies_to_target_matrikel og applies_to_matrikler fra input til at bestemme scope.
- Hvis applies_to_target_matrikel = true → scope = "Ja"
- Hvis applies_to_target_matrikel = false → scope = "Nej"
- Hvis applies_to_target_matrikel = null og applies_to_matrikler er tom → scope = "Måske"
- OBS — historiske akter: applies_to_matrikler kan indeholde GAMLE matrikelnumre (fx "1a", "1m") der ikke matcher ejendommens nuværende matrikler. Brug evidensteksten og servituttens indhold til at vurdere om det sandsynligvis vedrører projektområdet. Anfør din vurdering i scope_detail.
- Returner KUN valid JSON-objektet. Ingen tekst før eller efter JSON-objektet.
